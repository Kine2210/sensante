# Bilan Lab Git — Le Médecin

## 3 choses apprises dans les Labs Git

1. **Les branches** permettent de travailler en parallèle sans perturber le travail des autres membres de l'équipe.

2. **Les conflits Git** surviennent quand deux personnes modifient la même ligne du même fichier. Il faut lire les marqueurs (`<<<<<<<`, `=======`, `>>>>>>>`), choisir la bonne version, supprimer les marqueurs, puis faire `git add` + `git commit`.

3. **Le workflow complet** : créer une branche → coder → commit → push → ouvrir une Pull Request → L'Architecte review et merge → tout le monde fait `git pull`.
